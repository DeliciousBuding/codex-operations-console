# Codex Operations Console 视觉设计系统

更新时间：2026-03-26

> 主参考：外部 Fluent 2 设计资料

> 本文档是项目级 Fluent 2 落地规范。若临时判断或历史截图与本文冲突，以本文为准。

## 1. 目标

本项目的视觉目标不是“泛用后台模板”，而是：

- 让这个产品像一个真正的 **Codex OAuth 运营控制台**
- 把 **Windows 11 / Fluent 2 家族** 转译成适合 Web 控制台的视觉语言
- 让用户一眼觉得“这和我熟悉的 Windows 系统一家人”

这份文档是项目级视觉基线，后续任何 UI 改动、页面扩展、组件新增，都应优先遵守这里的规则，而不是临时拼样式。

---

## 2. 总体视觉定位

### 2.1 产品气质

产品应呈现：

- 现代
- 轻盈
- 系统感强
- 专业但不冰冷
- 控制台感明确
- 系统伴侣气质

避免：

- 通用后台模板味
- 卡片堆叠式“功能陈列台”
- 过度炫技的装饰感
- 过多同权按钮、同权标签、同权统计
- 随机网页感
- 与 Fluent 2 家族冲突的多风格混搭

### 2.2 页面角色分工

不同页面必须有不同语气：

- **概览页**：`Command Center / Situation View`
  - 回答“哪里在出事”
  - 回答“先做什么”
  - 回答“下一步入口在哪”

- **账号页 / 清理页**：`Workspace / Tool View`
  - 强调筛选、搜索、批量处理、表格操作
  - 视觉上更紧凑、更克制

- **设置页**：`System / Session View`
  - 信息简洁、参数明确
  - 少装饰，重可读

## 3. 主题策略

## 3.1 Fluent 2 在本项目中的角色

Fluent 2 在本项目中不是“灵感参考”，而是**主家族语言**。

需要保留的微软体系视觉 DNA：

- 柔和渐变背景
- Mica / Acrylic 材质层次
- 明确轮廓与稳定圆角
- Fluent 蓝系主色
- 轻量空间变化交互
- 系统级、温和、可信赖的整体气质

需要做的 Web 化适配：

- 保留 DNA，但避免直接抄桌面 UI
- 优先考虑云控制台 / 运营后台 / 效率工具的真实密度与信息层级
- 允许更强的表格、筛选、批量操作结构
- 允许更高的信息密度，但不破坏家族语言

## 3.2 Dark Theme：OLED 优化的暗色模式

Dark 主题是当前主主题，现已针对 OLED 显示器优化：

### OLED 优化原则

- **真正的黑色背景**：`#000000` 纯黑，最大化 OLED 能效和对比度
- **降低对比度**：主文字 `rgba(255,255,255,0.92)`，避免纯白刺激眼睛
- **微妙的灰色层级**：`#121212` 卡片 / `#1E1E1E` 三级背景
- **柔和的强调色**：`#6B8AFF` 蓝色系，减少视觉疲劳
- **纯黑阴影**：`rgba(0,0,0,0.5)` 增强立体感

### Dark 视觉关键词

- OLED optimized
- true black
- subtle contrast
- eye-friendly
- depth layers
- system panel
- command center

### Dark 应用规则

- 背景：纯黑 `#000000`（OLED 最优）
- 卡片：`#121212` 实心背景，`backdrop-filter: blur(16px)` 保持材质感
- 文字：降低对比度，主文字 92% 不透明度，次级文字 60%
- 选中态：`rgba(107,138,255,0.15)` 背景，`#6B8AFF` 主色
- 告警：`#FF5252` 红色系，降低饱和度
- Hover：`rgba(255,255,255,0.1)` 微妙白色叠加
- 过渡：全局 `0.2s ease-in-out` 平滑动画

### 与 Fluent 2 的差异

相比传统 Fluent 2 Dark：
- 移除渐变背景，改用纯黑
- 降低文字对比度（不再是 `#eff5fc` 纯白）
- 简化材质层，用实色代替半透明叠加
- 优化滚动条：`#6B8AFF` 主色，`#1E1E1E` 轨道

> 参考：外部 Dark Mode 设计资料

## 3.3 Light Theme：Fluent 2 Light / 柔和系统面板

Light 主题不是“苹果拟态”或“纯白后台”，而是 **Fluent 2 Light 在 Web 场景下的轻量版本**：

- 白色与浅灰蓝为主
- 保留柔和渐变背景
- 卡片仍然保留 Acrylic / 半透明 / 高光边界感
- 通过浅色阴影和高光体现系统层次，而不是靠重拟态压出体积

### Light 视觉关键词

- fluent light
- mica light
- acrylic panel
- soft depth
- calm system surface

### Light 应用规则

- 背景：白到浅灰的柔和渐变
- 卡片：半透明浅色底 + 柔和阴影 + 高光描边
- 按钮：更像系统按钮，而不是拟态按压块
- 状态色：少量点到为止，避免彩色块泛滥
- 交互：保持 Fluent 的稳定、系统级反馈

## 4. 色彩与层级规则

### 4.1 主色体系

- 主色：Fluent 蓝 / Apple 蓝
- 告警：红色仅用于异常中心、P0、严重风险
- 警告：橙色仅用于 warning / 次级风险
- 成功：绿色仅用于健康态 / Active

### 4.2 一屏原则

一屏里只能有：

- 1 个主强调色
- 1 个真正的主按钮
- 1 个真正的高优先级异常块

避免同屏出现多个“都像主角”的彩色元素。

---

## 5. 材质与控件规则

### 5.1 卡片

- Dark：Acrylic 卡片，细描边，低强度阴影，微高光边缘
- Light：Fluent Light 卡片，浅色半透明底 + 柔和阴影 + 内外高光
- 普通卡片与高危卡片必须区分，但不能全部做成“重卡”
- 卡片 header / body / footer 必须继承统一圆角
- 禁止再出现部分卡片方角、部分卡片圆角的情况

### 5.2 按钮

按钮优先级固定：

1. **Primary CTA**
2. **Secondary button**
3. **Ghost / text tool**

规则：

- 每个区域只允许 1 个真正主按钮
- 次级按钮不要做得像主按钮
- ghost 工具按钮应该尽量安静
- Hover / Active 必须更接近系统控件，而不是网页 CTA 特效

### 5.3 标签 / 徽章

严格遵守：

- 一行只保留 **1 个强标签**
- 其余标签全部降级成弱标签或次级文字

典型优先级：

1. 风险 / 异常
2. 状态
3. 分层 / 分组

不要让 `plan / group / status / issue / cleanup` 同时都用高饱和 badge。

---

## 6. 信息架构规则

### 6.1 概览页

概览页必须优先展示：

1. 当前最高优先级问题
2. 当前异常规模
3. 当前第一步操作入口

不允许再退化成“平均铺开的 dashboard”。

### 6.2 账号页

账号页层级顺序固定：

1. 一级视图切换：`全部 / Team / Free / Cleanup / Disabled`
2. 搜索
3. 二级筛选：自定义组
4. 表格
5. 批量栏（仅在选中时出现）

动作按钮不能和筛选项混成同一层。

### 6.3 侧栏

侧栏只做三件事：

1. 品牌识别
2. 主导航
3. 轻量环境上下文

侧栏底部 context 不应变成第二个信息面板。

同时必须满足：

- 图标与文字关系稳定
- 不允许出现文字被遮挡
- 选中态要像 Windows 11 控制面板中的当前项，而不是网页 tab
- 导航项要足够轻，但仍有系统控件感

---

## 7. 动效与交互

统一规则：

- 时长：`150–300ms`
- 曲线：平滑，不弹跳
- Hover：亮度 / 边框 / 阴影 / 轻上浮的细微变化
- Active：轻微下沉或收紧
- 进度、切换、状态变化优先采用柔和宽度变化或透明度变化

禁止：

- 大幅位移
- 夸张弹跳
- 过亮玻璃高光
- 高频动态背景

## 9. 后续开发守则

后续任何页面或组件新增前，先问：

1. 这个页面属于 `Command Center` 还是 `Workspace`？
2. 当前屏幕的唯一主按钮是谁？
3. 当前屏幕的唯一强异常块是谁？
4. 这行里是否出现了超过 1 个高饱和标签？
5. 这个信息是否已经在别处出现过，是否属于重复广播？
6. 这是不是还能让用户感觉“这和 Windows 11 / Fluent 2 是一家人”？
7. 这是不是已经被网页组件库默认风格带偏了？

只要有任意一条答不上来，就不应该继续直接写样式。
