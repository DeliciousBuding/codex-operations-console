# 文档索引

这个目录存放 `codex-operations-console` 的设计、验证和优化文档。

如果你是第一次查看这个项目，建议按下面顺序阅读。

## 推荐阅读顺序

1. `../README.md`
   仓库首页，介绍项目目标、能力边界和开发方式。

2. `visual-design-system.md`
   项目的设计基线。涉及 UI、视觉层级、主题或组件风格时，优先看这份文档。

3. `browser-test-workflow.md`
   浏览器测试和页面复核流程。

4. `multimodal-review-workflow.md`
   更完整的多模态审查方法，适合继续做视觉迭代时参考。

## 文档分组

### 核心文档

- `visual-design-system.md`
  视觉设计系统与页面角色定义。

- `browser-test-workflow.md`
  浏览器测试与截图复核流程。

- `multimodal-review-workflow.md`
  多模态审查方法说明。

- `frontend-optimization-summary.md`
  当前前端优化方向与核心改进摘要。

- `p0-fixes-report.md`
  关键问题修复摘要。

### 阶段性报告

这些文档记录历史阶段的评审、验证和优化结果，适合需要追踪过程时查阅：

- `archive/`
  阶段性报告、优化记录和历史留档入口。

### 截图资源

- `screenshots/`
  公开保留的示例截图，用于 README 和文档展示。

## 维护原则

- 对外优先保留“结果型文档”，减少过程噪音
- 设计和测试入口应稳定，避免重复说明
- 涉及私有部署、服务器信息或敏感环境内容，不应写入这里
