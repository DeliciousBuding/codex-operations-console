# 发布说明

这份文档用于说明 `codex-operations-console` 的版本发布方式。📦

## 🎯 发布目标

项目当前采用轻量发布策略：

- 以 Git 提交和 GitHub Release 作为版本记录
- 以 `README.md` 和变更说明作为主要对外沟通入口
- 不引入复杂发布流水线，优先保证内容清晰和仓库干净

## 🏷️ 版本建议

推荐采用语义化版本：

- `v0.x.y`
- `v1.x.y`

当前阶段更适合：

- `v0.1.0` 初始公开版本
- `v0.2.0` 新增明显功能或结构改进
- `v0.2.1` 小修复、小幅文档更新或非破坏性优化

## ✅ 发布前检查

发布前建议至少完成以下核对：

```powershell
npm run lint
npm test
npm run type-check
npm run build
```

同时确认：

- `README.md` 已反映当前项目状态
- 未提交任何敏感信息
- 若涉及 UI 变化，已补充截图或更新文档

## 📝 Release Notes 建议结构

每次发布可按以下结构整理说明：

### 新增
- 新页面
- 新交互
- 新文档

### 改进
- 样式优化
- 可读性优化
- 结构调整

### 修复
- Bug 修复
- 类型修复
- 构建或依赖修复

### 已知限制
- 暂未覆盖的场景
- 后续计划

## 🚀 GitHub 发布建议

推荐流程：

1. 确认 `main` 已是待发布状态
2. 创建 tag，例如 `v0.1.0`
3. 在 GitHub 创建对应 Release
4. 填写版本说明与截图

## 🔐 发布边界

发布前请再次确认：

- 不包含私有部署入口
- 不包含真实 token、cookie 或环境密文
- 不包含本机敏感路径或内部运维说明

公开仓库只负责源码、公开文档和通用协作内容。
